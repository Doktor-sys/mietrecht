# Mobile App Publication Checklist

## Voraussetzungen

- [ ] Expo Account erstellt und eingerichtet
- [ ] Apple Developer Account (für iOS)
- [ ] Google Play Developer Account (für Android)
- [ ] EAS CLI installiert (`npm install -g eas-cli`)
- [ ] Projekt-Abhängigkeiten installiert (`npm install`)

## App-Konfiguration

- [ ] `app.json` mit korrekten App-Metadaten
- [ ] Bundle Identifier für iOS: `com.smartlaw.mietrecht`
- [ ] Package Name für Android: `com.smartlaw.mietrecht`
- [ ] App Icons und Splash Screens erstellt und platziert
- [ ] Berechtigungen für Kamera und Dateizugriff konfiguriert
- [ ] Deep Linking für beide Plattformen eingerichtet
- [ ] Projekt-ID in `app.json` aktualisiert

## EAS-Konfiguration

- [ ] `eas.json` mit Build-Profilen (development, preview, production)
- [ ] iOS Build-Konfiguration überprüft
- [ ] Android Build-Konfiguration überprüft
- [ ] Submit-Konfiguration für App Store und Play Store

## Assets

- [ ] App Icon (1024x1024) für iOS
- [ ] Adaptive Icon für Android
- [ ] Splash Screen
- [ ] Screenshots für App Store (iPhone, iPad)
- [ ] Screenshots für Play Store (Phone, Tablet)
- [ ] Feature Graphic für Play Store
- [ ] Promotional Text für beide Stores

## App Store Connect (iOS)

- [ ] App in App Store Connect erstellt
- [ ] App-Informationen ausgefüllt
- [ ] Metadata (Beschreibung, Keywords, Datenschutz) bereitgestellt
- [ ] Altersfreigabe beantragt
- [ ] Preis und Verfügbarkeit festgelegt
- [ ] Kontaktinformationen hinterlegt

## Google Play Console (Android)

- [ ] App in Google Play Console erstellt
- [ ] Store-Eintrag ausgefüllt
- [ ] App-Beschreibung und Grafiken hochgeladen
- [ ] Content Rating durchgeführt
- [ ] Kontaktinformationen hinterlegt
- [ ] Datenschutzrichtlinie verlinkt

## Sicherheit

- [ ] API-Schlüssel in Umgebungsvariablen
- [ ] HTTPS für alle API-Anfragen
- [ ] Sichere Token-Verwaltung
- [ ] Verschlüsselung sensibler Daten

## Testing

- [ ] Funktionstests auf verschiedenen Geräten
- [ ] UI-Tests auf verschiedenen Bildschirmgrößen
- [ ] Performance-Tests
- [ ] Sicherheitstests
- [ ] Beta-Tests mit ausgewählten Benutzern

## Build und Deployment

- [ ] Development Build erfolgreich erstellt
- [ ] Preview Build erfolgreich erstellt
- [ ] Production Build erfolgreich erstellt
- [ ] iOS Build an App Store Connect übermittelt
- [ ] Android Build an Google Play Console übermittelt

## Nach Veröffentlichung

- [ ] App-Performance überwachen
- [ ] Benutzerbewertungen und Feedback sammeln
- [ ] Fehlerberichte analysieren
- [ ] Marketing und Promotion planen
- [ ] Roadmap für zukünftige Updates erstellen
