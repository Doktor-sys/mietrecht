# Task Management Cheat Sheet

Quick reference for the new task management system

## Task Format

```
- [ ] **Task Title** (Prio: High | Est: 3 days | Owner: John | Depends on: Previous Task):
    - Brief description of what needs to be done
    - Acceptance Criteria (for complex tasks):
        - Specific, measurable outcome 1
        - Specific, measurable outcome 2
```

## Status Indicators

| Symbol | Status      | When to Use                     |
|--------|-------------|---------------------------------|
| [x]    | Completed   | Task finished and verified      |
| [~]    | In Progress | Actively being worked on        |
| [ ]    | Planned     | Scheduled for future work       |
| [!]    | Blocked     | Cannot progress due to blocker  |
| [?]    | Need Info   | Requires clarification          |

## Priority Levels

| Level    | Use For                                | Examples                        |
|----------|----------------------------------------|---------------------------------|
| Critical | Immediate work; blocks other tasks     | Security issues, core features  |
| High     | Important for current goals            | Feature development, bug fixes  |
| Medium   | Normal day-to-day work                 | Tech debt, documentation        |
| Low      | Nice-to-have improvements              | UI tweaks, minor enhancements   |

## Daily Standup Format

Report on:
1. **Yesterday** - What you accomplished
2. **Today** - What you plan to work on
3. **Blockers** - Anything preventing progress

Example:
- Yesterday: Finished authentication service tests
- Today: Starting document storage integration
- Blockers: None

## Weekly Grooming Checklist

□ Review task priorities
□ Update effort estimates
□ Identify and resolve blockers
□ Assign new tasks
□ Review dependencies

## Creating New Tasks

1. **Title** - Specific and action-oriented
2. **Priority** - Based on business impact
3. **Estimate** - Realistic effort in days
4. **Owner** - Person responsible
5. **Dependencies** - What must be completed first
6. **Description** - Clear explanation of work
7. **Acceptance Criteria** - For complex tasks
