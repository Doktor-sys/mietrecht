# Asana Quick Start Guide

This guide provides a quick introduction to using Asana for our task management.

## Getting Started

### 1. Account Setup
- Check your email for an invitation from Asana
- Click the invitation link to create your account
- Set up your profile with your name and photo
- Download the mobile app for on-the-go access

### 2. Navigating Asana
- **My Tasks**: Your personal task list
- **Projects**: Team projects organized by area
- **Portfolios**: Collections of related projects
- **Calendar**: View tasks by due date
- **Reports**: Track progress and performance

## Creating Tasks

### Basic Task Creation
1. Click the "+" button in the top bar
2. Select "Task"
3. Enter a descriptive task name
4. Add to a project (optional but recommended)
5. Set a due date (if applicable)
6. Assign to yourself or a team member
7. Click "Create Task"

### Detailed Task Setup
For complex tasks, add these details:
- **Description**: Detailed explanation of what needs to be done
- **Assignee**: Who is responsible for completing the task
- **Due Date**: When the task should be completed
- **Project**: Which project the task belongs to
- **Priority**: Use custom priority field (Critical, High, Medium, Low)
- **Estimate**: How long the task is expected to take (in days)
- **Dependencies**: Tasks that must be completed first

## Task Management

### Updating Task Status
- Move tasks between workflow stages (Planned, In Progress, In Review, Blocked, Completed)
- Update progress percentage
- Add comments to communicate with team members
- Mark subtasks as complete

### Collaboration Features
- **Comments**: Discuss task details with team members
- **Attachments**: Add files, images, or documents
- **Tags**: Categorize tasks with custom tags
- **Followers**: Add people who need to be notified about task updates

## Best Practices

### Task Naming
- Use action verbs: "Implement user authentication" rather than "User authentication"
- Be specific: "Create database schema for user profiles" rather than "Database work"
- Keep concise but descriptive

### Descriptions
- Include enough detail for anyone to understand what needs to be done
- Link to relevant documents, designs, or specifications
- Specify acceptance criteria clearly

### Assigning Tasks
- Assign tasks to the person responsible for completion
- Use " Followers" for people who need to be kept informed
- Don't overload team members with too many assigned tasks

### Due Dates
- Set realistic deadlines
- Use due dates to communicate priorities
- Update due dates when circumstances change

### Communication
- Use comments to discuss task details rather than email
- @mention team members to get their attention
- Keep communication in Asana for transparency

## Common Workflows

### Starting a New Task
1. Find or create the task in Asana
2. Move it to "In Progress" status
3. Add any initial comments or updates
4. Begin work on the task

### Completing a Task
1. Ensure all acceptance criteria are met
2. Move task to "In Review" status
3. Add a comment summarizing what was done
4. Assign to reviewer if needed
5. Once approved, move to "Completed" status

### Getting Blocked
1. Move task to "Blocked" status
2. Add a comment explaining what's blocking progress
3. @mention the person who can help unblock
4. Update the task when unblocked

## Quick Reference

### Keyboard Shortcuts
- **Tab + /**: Quick add task
- **Tab + N**: Create new task
- **Tab + Z**: Mark task as complete
- **Up/Down arrows**: Navigate tasks
- **Enter**: Open task
- **B**: Add to today
- **D**: Set due date

### Status Indicators
- **Planned**: Task identified but not yet started
- **In Progress**: Actively being worked on
- **In Review**: Completed but awaiting review/validation
- **Blocked**: Cannot progress due to external factors
- **Completed**: Fully finished and accepted
- **Cancelled**: No longer needed

### Priority Levels
- **Critical**: Must be done immediately; blocks other work
- **High**: Important for current goals; should be done soon
- **Medium**: Part of regular workflow
- **Low**: Nice to have; can be deferred
