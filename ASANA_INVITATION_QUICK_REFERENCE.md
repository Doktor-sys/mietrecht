# Asana Invitation Process - Quick Reference

This document provides a quick reference for the Asana invitation process.

## Quick Steps

### 1. Prepare
- [ ] Gather team member email addresses
- [ ] Determine appropriate roles (Admin/Member/Guest)
- [ ] Review ASANA_TEAM_INVITATIONS.md

### 2. Send Invitations
- [ ] Log into Asana as Admin
- [ ] Navigate to "Add Members"
- [ ] Enter email addresses
- [ ] Assign roles
- [ ] Send invitations

### 3. Track Responses
- [ ] Monitor ASANA_INVITATION_TRACKING.md
- [ ] Send reminders after 24 hours
- [ ] Follow up after 48 hours
- [ ] Final follow-up after 72 hours

### 4. Resolve Issues
- [ ] Help with login problems
- [ ] Resend invitations as needed
- [ ] Update tracking document
- [ ] Escalate persistent issues

## Key Information

### Roles
- **Admin**: Full organization access, can manage settings
- **Member**: Can create/manage tasks, assigned to projects
- **Guest**: Limited access, only to specific projects

## Common Issues & Solutions

### Invitation Not Received
1. Check spam/junk folder
2. Verify email address is correct
3. Resend invitation from Asana
4. Contact IT to check email filtering

### Can't Accept Invitation
1. Ensure using correct email address
2. Try different browser
3. Clear browser cache and cookies
4. Contact Asana support

### Wrong Role Assigned
1. Admin can change role in "Admin Console" > "Users"
2. Remove user and re-invite with correct role
3. Contact support if issues persist

### Can't Access Projects
1. Verify user was added to project
2. Check project privacy settings
3. Confirm user role permissions
4. Add user to project manually
